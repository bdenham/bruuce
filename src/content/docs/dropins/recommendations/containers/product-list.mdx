The `ProductList` container manages and displays a list of recommended products based on the current product or category context. Its behavior is driven by configuration options such as currentSku, pageType and routing via routeProduct callbacks.

<Diagram caption="ProductList container">
  ![ProductList container](@images/dropins/recommendations/product-list.png)
</Diagram>

## Configurations

The `ProductList` container provides the following configuration options:

<OptionsTable
  compact
  options={[
    ['Option', 'Type', 'Req?', 'Description'],
    ['currentSku', 'string', 'Yes', 'Current SKU for recommendation context.'],
    ['cartSkus', 'string', 'array', 'Cart SKUs for recommendation context.'],
    ['category', 'string', 'No', 'Category for recommendation context.'],
    ['userPurchaseHistory', 'array', 'No', 'User Purchase History for recommendation context.'],
    ['userViewHistory', 'array', 'No', 'User View History for recommendation context.'],
    ['routeProduct', 'function', 'No', 'Callback function that returns a product URL.'],
    ['pageType', 'function', 'No', 'Page type where recommendations are rendered (e.g., Product, Cateogy, Cart)'],
    ['className', 'function', 'No', 'Custom CSS class for styling.'],
  ]}
/>



## Example configuration

The following example demonstrates how to render the `ProductList` container with the `currentSku`,`pageType` and `routeProduct` callbacks:

```js

          provider.render(ProductList, {
            routeProduct: (item) => {
              return `${item.urlKey}`;
            },
            currentSku: 'CTEAPB',
            pageType: 'Product'
          })(document.getElementById('@dropins/ProductList'));
```
