The WishlistItem container manages and displays a single product of a wishlist, allowing users to interact with it, such as moving the product to the cart or navigating to the product details page.
In case of complex products with no variant selected in a wishlist, the move to cart button is disabled, and the user is prompted to select a variant first by navigating to the product details page.
Its behavior is driven by configuration options such as routeEmptyWishlistCTA, routeEmptyWishlistCTA and routing via routeProdDetailPage callbacks.

## Configurations

The `WishlistItem` container provides the following configuration options:

## Example

The following example demonstrates how to render the `WishlistItem` container:

```javascript
provider.render(WishlistItem, {
    initialData: sampleWishlistItem,
    moveProdToCart: cartApi.addProductsToCart,
    routeProdDetailPage: (product) => rootLink(`/products/${product.urlKey}/${product.sku}`),
  });
```
