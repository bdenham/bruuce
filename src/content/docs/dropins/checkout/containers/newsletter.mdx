The `Newsletter` container displays a checkbox that allows users to subscribe to the latest news and events updates during a guest checkout process. The subscription will be stored when the user clicks the **Place Order** button.

## Newsletter configurations

The `Newsletter` container provides the following configuration options:

<OptionsTable
  compact
  options={[
    ['Option', 'Type', 'Req?', 'Description'],
    ['defaultChecked', 'boolean', 'No', 'Displays the checkbox checked by default (default value is false).'],
    ['onChange', 'function', 'No', 'Callback function that is called when the checkbox state changes.'],
    ['showForAuthenticatedUsers', 'boolean', 'No', 'Displays the container even if the user has signed-in (default value is false).'],
  ]}
/>

These configuration options are implementing the `NewsletterProps` interface:

### NewsletterProps interface

The `Newsletter` container receives an object as a parameter which implements the `NewsletterProps` interface with the following properties:

```ts
export interface NewsletterProps {
  defaultChecked?: boolean;
  onChange?: (checked: boolean) => void;
  showForAuthenticatedUsers?: boolean;
}
```

<li>The `defaultChecked` property set to _true_ displays the checkbox checked to subscribe the user by default.</li>

<li>The `onChange` property is a handler used to perform actions called when the checkbox is checked/unchecked.</li>

<li>The `showForAuthenticatedUsers` property set to _true_ displays the `Newsletter` container to allow authenticated users to handle the subscription.</li>

## Example 1: Render the default container

The following example renders the `Newsletter` container on a checkout page.

```ts
import Newsletter from '@dropins/storefront-checkout/containers/Newsletter.js';
import { render as CheckoutProvider } from '@dropins/storefront-checkout/render.js';

const $newsletter = checkoutFragment.querySelector('.checkout__newsletter');

CheckoutProvider.render(Newsletter, {})($newsletter),
```

## Example 2: Render the subscription checked by default

The following example renders the `Newsletter` container on a checkout page, checked by default.

```ts
import Newsletter from '@dropins/storefront-checkout/containers/Newsletter.js';
import { render as CheckoutProvider } from '@dropins/storefront-checkout/render.js';

const $newsletter = checkoutFragment.querySelector('.checkout__newsletter');

CheckoutProvider.render(Newsletter, {
  defaultChecked: true,
})($newsletter),
```

## Example 3: Render the subscription for authenticated users

The following example renders the `Newsletter` container even if the user has been authenticated.

```ts
import Newsletter from '@dropins/storefront-checkout/containers/Newsletter.js';
import { render as CheckoutProvider } from '@dropins/storefront-checkout/render.js';

const $newsletter = checkoutFragment.querySelector('.checkout__newsletter');

CheckoutProvider.render(Newsletter, {
  showForAuthenticatedUsers: true,
})($newsletter),
```

## Example 4: Render the subscription providing a function to handle state changes

The following example renders the `Newsletter` container doing "something" when the checkbox is checked/unchecked.

```ts
import Newsletter from '@dropins/storefront-checkout/containers/Newsletter.js';
import { render as CheckoutProvider } from '@dropins/storefront-checkout/render.js';

const $newsletter = checkoutFragment.querySelector('.checkout__newsletter');

CheckoutProvider.render(Newsletter, {
  onChange: (checked) => {
    if (checked) {
      // Do something when checkbox is checked
    } else {
      // Do something when checkbox is unchecked
    }
  },
})($newsletter),
```
